# Social Content Generator 使用教程

## 📖 快速入门

### 基础用法

```
用户提供链接 → AI 自动生成完整社媒内容包
```

**示例对话**:

```
用户: https://github.com/ItzCrazyKns/Perplexica

AI: 
✅ 已抓取内容：AI 搜索引擎 Perplexica
✅ 生成推文文案 (3 版本)
✅ 推荐 SEO 标签 (按平台分类)
✅ 设计系列封面 (3 张一套，默认设计工具风)
✅ 吸睛标题 (5 个选项)
✅ 一句话描述 (3 个版本)
```


**垂类型** (精准受众):

#### 3.3 系列封面 (3 张一套)

- 标题: "你的前端代码"

- 视觉: 混乱的设计稿、红色错误标记
- 情绪: 焦虑、困扰

**封面 2 - 方案篇** 😃
- 标题: "Impeccable"
- 副标题: "Anthropic 前端设计技能升级版"

**封面 3 - 行动篇** 🚀
- 标题: "告别设计债"
- 副标题: "让代码配得上你的设计"
- 视觉: 火箭发射、成功场景
- 情绪: 兴奋、行动

| 类型 | 标题 |
|------|------|

| 结果型 | 靠这个工具，我每天少加班 2 小时 |

- **功能导向**: "Impeccable 是基于 Anthropic 原始前端设计技能的升级版..."
- **利益导向**: "用 Impeccable，5 分钟修复所有设计债..."

## 🎨 自定义风格

### 选择视觉风格

```
用户: https://example.com --style=cyberpunk
```

**可选风格**:

| 风格 | 效果 | 适用场景 |
|------|------|---------|
| `design-tool` | Figma 紫/粉渐变 | 设计工具、创意产品 |
| `geek-code` | Terminal 深色 | 开发者工具、API |
| `minimal-card` | 黑白灰极简 | 高端产品、咨询服务 |
| `warm-life` | 暖橙渐变 | 生活方式、副业变现 |
| `cyberpunk` | 霓虹蓝紫 | AI 产品、前沿科技 |
| `business-pro` | 深蓝金边 | 企业服务、金融 |

### 选择目标平台

```
用户: https://example.com --platform=xiaohongshu
```

**可选平台**:

| 平台 | 特点 | 最佳发布时间 |
|------|------|-------------|
| `twitter` / `x` | 简洁犀利 | 8:00-9:00, 17:00-18:00 |
| `xiaohongshu` | emoji+标签 | 20:00-22:00 |
| `jike` | 轻松幽默 | 12:00-14:00 |
| `linkedin` | 专业洞察 | 工作日白天 |

### 组合参数

```
用户: https://example.com --style=minimal-card --platform=linkedin --tone=professional
```

**完整参数列表**:

| 参数 | 说明 | 默认值 | 可选值 |
|------|------|--------|--------|
| `--style` | 视觉风格 | design-tool | design-tool, geek-code, minimal-card, warm-life, cyberpunk, business-pro |
| `--platform` | 目标平台 | xiaohongshu | twitter, xiaohongshu, jike, linkedin |
| `--cover` | 封面数量 | 3 | single, 3 |
| `--tone` | 文案语调 | casual | professional, casual, bold |
| `--lang` | 输出语言 | zh | zh, en |

## 📱 平台发布指南

### 小红书

**最佳实践**:
- 发布时间: 20:00-22:00
- 封面: 3 张轮播 (问题→方案→行动)
- 文案: 故事版 (情绪共鸣强)
- 标签: 8-12 个 (含 #干货分享 #宝藏app)

**发布步骤**:
1. 上传 3 张封面图
2. 粘贴故事版文案
3. 添加标签 (复制推荐标签)
4. 首评补充: "需要安装教程的评论区扣 1"

### Twitter/X

**最佳实践**:
- 发布时间: 8:00-9:00 (早高峰)
- 封面: 单张 (行动篇)
- 文案: 钩子版 (简洁犀利)
- 标签: 3-5 个英文标签

**Thread 技巧**:
```
推文 1: 钩子 + 封面图
推文 2: 核心功能 (3-5 点)
推文 3: 使用场景
推文 4: CTA + 链接
```

### 即刻

**最佳实践**:
- 发布时间: 12:00-14:00 (午休)
- 封面: 单张 (方案篇)
- 文案: 干货版 (信息密度高)
- 标签: 5-8 个 (圈子标签)

**圈子选择**:
- 产品经理
- 独立开发者
- 效率工具
- 设计沉思录

### LinkedIn

**最佳实践**:
- 发布时间: 工作日 9:00-11:00
- 封面: 单张 (商务专业风)
- 文案: 专业版 (行业洞察)
- 标签: 3-5 个专业标签

**内容结构**:
```
Hook (吸引注意)
↓
Story/Context (背景故事)
↓
Key Insights (核心观点)
↓
Call to Action (行动号召)
↓
Hashtags (3-5个)
```

## 💡 高级技巧

### A/B 测试标题

同时准备 3-5 个标题，测试点击率:

```
标题 A: 数字型 → 点击率 5.2%
标题 B: 疑问型 → 点击率 8.7% ✅
标题 C: 警告型 → 点击率 6.1%
```

**结论**: 使用疑问型标题

### 优化标签策略

**基础公式**:
```
2个品牌标签 + 4个垂类标签 + 4个流量标签
```

**追踪指标**:
- 曝光量
- 点击率
- 互动率
- 转化率

### 封面设计原则

**安全区域**:
```
顶部 15% - 平台 UI 遮挡区
中间 73% - 核心内容安全区
底部 12% - 描述文字遮挡区
```

**文字层级**:
- 主标题: 占画面 15-20%
- 副标题: 占画面 8-10%
- 账号标识: 底部固定位置

## 🔧 故障排除

### 常见问题

**Q: 生成的文案太长了怎么办？**
A: 使用 `--platform=twitter` 参数，自动压缩到 280 字符内。

**Q: 封面风格不喜欢怎么办？**
A: 使用 `--style=` 参数切换风格，如 `--style=cyberpunk`。

**Q: 标签不够精准怎么办？**
A: 在参考文件中查看 `references/seo-tags.md`，手动选择更精准的标签。

**Q: 如何生成英文内容？**
A: 使用 `--lang=en` 参数。
