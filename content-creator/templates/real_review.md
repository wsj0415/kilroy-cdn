# 实战测评型模板

## 适用场景
- 深度使用过的工具
- 有真实使用数据
- 能说出优缺点

## 核心原则
- **真实体验**：只说实际用过的
- **优缺点都说**：更可信
- **数据支撑**：有时间/效果数据
- **适用人群**：帮助决策

## 文案结构

```markdown
【标题】[痛点]？这个 [工具类型] 真香/救了命

【开头】
[使用场景] 后，我发现了 [工具名]。
先说结论：[一句话评价]

【真香功能】
🔥 实测 [N] 个真香功能：
1. [功能名]
   以前：[痛点]
   现在：[改善]

【注意事项】
⚠️ [N] 个注意事项（实话实说）：
1. [注意点]
   [详细说明]

【适用人群】
💡 适合谁用：
✅ [适合场景 1]
✅ [适合场景 2]

❌ 不适合谁：
[不适合场景]

【项目信息】
🔗 项目信息：
GitHub：[链接]
在线体验：[链接]
技术栈：[技术栈]

【总结】
总的来说，[真实评价]。
有同样困扰的 [目标用户]，可以试试。

【标签】
#[标签 1] #[标签 2] ...（10 个）
```

## 示例

```markdown
受够了每个 AI 工具单独配 skills？这个开源项目真香

装了 3 个 AI 编程助手后，我发现了 Skills Manager。
先说结论：多工具用户必备，单工具用户没必要。

🔥 实测 3 个真香功能：
1. 批量启用/禁用
   以前：每个工具单独配，重复劳动
   现在：一键切换，效率提升 80%

2. GitHub 安装
   看到好用的 skill，贴链接就能装
   不用手动下载配置

3. 跨工具复制
   Cursor 的 skills 一键复制到 Claude Code
   这个真的省时间

⚠️ 2 个注意事项（实话实说）：
1. 需要 Electron 环境
   第一次配置需要一些时间
   建议官方出个一键安装包

2. 部分 skills 兼容性需要测试
   建议先备份原有配置
   再尝试迁移

💡 适合谁用：
✅ 装了 3 个以上 AI 编程工具
✅ 经常折腾新 skills
✅ 需要统一管理配置

❌ 不适合谁：
只用 1-2 个工具，手动配就行
不想装额外软件的

🔗 项目信息：
GitHub：github.com/zunalabs/skills-manager
在线体验：sm.idoevergreen.me
技术栈：Electron + Next.js + TypeScript

总的来说，这项目解决了一个真实痛点。
虽然有点门槛，但用起来是真方便。
有同样困扰的开发者，可以试试。

#人工智能 #AI 编程 #开源项目 #效率工具 #开发者 #GitHub #编程助手 #技术前沿 #程序员 #VSCode
```
