# 资讯分享型模板

## 适用场景
- 新发现的开源项目
- 没时间实测的工具
- 基于公开信息的介绍

## 核心原则
- 实事求是，不编造体验
- 有什么说什么
- 不确定就直说
- 把判断权给用户

## 文案结构

```markdown
【标题】发现一个 [领域] 项目，[核心功能]

【开头】
项目叫 [名称]，刚看到的。

【功能信息】
📊 它能做什么：
- 功能 1
- 功能 2
- 功能 3

【技术信息】
🔍 技术信息：
- 技术栈
- 开源地址
- 在线预览

【解决问题】
💡 解决什么问题：
[目标用户] 的 [具体痛点]

【个人看法】
📌 个人看法：
- 项目状态（新/成熟）
- 没实际使用过，不知道 [某方面]
- 思路是对的，[某痛点] 确实需要解决
- 感兴趣的可以自己试试

【链接】
🔗 链接：
GitHub：[链接]
在线体验：[链接]

【标签】
#[标签 1] #[标签 2] ...（10 个）
```

## 示例

```markdown
发现一个开源项目，统一管理 11 个 AI 编程助手的 skills

项目叫 Skills Manager，刚看到的。

📊 它能做什么：
统一管理 Claude Code、Cursor、Copilot、Gemini CLI 等 11 个工具的 skills
用一个界面浏览、启用、禁用、安装、删除
支持从任意 GitHub 仓库安装 skills
可以在不同工具之间复制 skills

🔍 技术信息：
- 桌面应用：Electron + Vite + React + TypeScript
- 开源地址：github.com/zunalabs/skills-manager
- 在线预览：sm.idoevergreen.me
- 技术栈：TypeScript + Tailwind CSS

💡 解决什么问题：
如果你装了多个 AI 编程工具（比如同时用 Cursor、Claude Code、Copilot）
每个工具单独配置 skills 很麻烦
这个工具想统一管理层

📌 个人看法：
- 项目比较新，Star 数不多（具体多少没查）
- 没实际使用过，不知道稳定性如何
- 思路是对的，多工具用户确实有这个痛点
- 感兴趣的可以自己试试，开源的随便折腾

🔗 链接：
GitHub：https://github.com/zunalabs/skills-manager
在线体验：https://sm.idoevergreen.me

有在用类似工具的朋友，可以分享一下体验。

#人工智能 #AI 编程 #开源项目 #效率工具 #开发者 #GitHub #编程助手 #技术前沿 #程序员 #VSCode
```

## 平台适配

### 微头条
- 字数：300-500 字
- 格式：每 2-3 句换行
- 标签：10 个

### 小红书
- 字数：800-1200 字
- 格式：emoji + 分段
- 标签：10-15 个

### 公众号
- 字数：1500+ 字
- 格式：深度内容
- 标签：5-8 个
